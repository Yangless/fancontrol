# 部署开机自启动任务

## 问题诊断结果

**根本原因：**
1. 缺少开机自启动任务 - 只有定时任务，开机后 FanControl 不会自动启动
2. 任务配置问题 - 定时任务调用脚本，但脚本只负责切换配置，不负责启动
3. 缺少 `-tray` 参数 - FanControl 启动时没有显示系统托盘图标

## 已完成的修改

✅ **已修改 `C:\FanControl_Auto\auto_switch.ps1`**
- 添加了 `-tray` 参数，确保启动时显示系统托盘图标
- 两处启动调用都已添加该参数

## 需要手动执行的步骤

### 步骤 1：以管理员身份运行部署脚本

```powershell
# 方法 1：右键 PowerShell 以管理员身份运行，然后执行：
powershell -ExecutionPolicy Bypass -File "C:\FanControl_Auto\deploy_startup.ps1"

# 方法 2：直接以管理员权限运行：
Start-Process powershell -Verb RunAs -ArgumentList '-ExecutionPolicy Bypass -File "C:\FanControl_Auto\deploy_startup.ps1"'
```

### 步骤 2：验证任务创建

```powershell
Get-ScheduledTask | Where-Object { $_.TaskName -like '*FanControl*' }
```

应该看到 5 个任务：
- FanControl-Startup (新增 - 开机自启动)
- FanControl-0800(Game)
- FanControl-1240(Quiet)
- FanControl-1400(Game)
- FanControl-2100(Quiet)

### 步骤 3：测试启动逻辑

```powershell
# 手动测试启动任务
powershell -ExecutionPolicy Bypass -File "C:\FanControl_Auto\auto_switch.ps1"
```

### 步骤 4：重启验证

重启系统后，检查：
1. FanControl 是否自动启动
2. 右下角是否显示风扇图表
3. 任务管理器中确认进程启动时间

## 预期效果

- 开机后自动启动 FanControl
- 根据当前时间自动选择配置（Game 或 Quiet）
- 系统托盘显示风扇监控图标
- 定时任务继续在特定时间点强制切换配置
