\# FanControl 自动化配置切换系统部署文档



\## 1. 架构概述与核心逻辑



本系统由两个 PowerShell 脚本和 Windows 任务计划程序组成，核心解决“自动切换配置”与“用户手动干预”之间的冲突问题。



\*\*官方特性利用：\*\*

利用 FanControl 的 `FanControl.exe -c <Config.json>` 命令。官方说明指出：如果后台已运行 FanControl，该命令不会开启新进程，而是\*\*发送进程间消息（IPC）让其无缝热切换\*\*。



\*\*时序与状态逻辑：\*\*

\* \*\*Game 时段\*\*：`08:00-12:40` 及 `14:00-21:00`

\* \*\*Quiet 时段\*\*：`12:40-14:00` 及 `21:00-次日08:00`

\* \*\*免打扰（手动）模式\*\*：用户通过热键/快捷方式主动切换配置时，生成 `override.flag` 标志，暂停后续的自动切换。

\* \*\*强制恢复机制\*\*：在进入 Quiet 时段的初始触发点（12:40 和 21:00），系统会\*\*强制清除免打扰标志\*\*，强制进入安静模式，并恢复自动调度。

\## 2. 准备工作与目录结构



1\. 打开 FanControl，提前保存好两个配置文件（建议纯英文命名）：`Quiet\_mode.json` 和 `Game\_ultr.json`。

2\. 建议在 C 盘根目录创建专用脚本文件夹：`C:\\FanControl\_Auto`。



\*\*最终文件结构：\*\*

```text

C:\\FanControl\_Auto\\

&#x20;├── auto\_switch.ps1      # 自动调度脚本（由系统定时执行）

&#x20;├── switch.ps1           # 手动干预脚本（绑热键/快捷方式用）

&#x20;└── state\\

&#x20;     └── override.flag   # 运行时自动生成的免打扰标志

```

\## 3. 核心代码实现



请在 `C:\\FanControl\_Auto` 目录下创建以下两个脚本。



\### 脚本 1：`auto\_switch.ps1` (自动调度后台)

此脚本由系统任务计划调用，负责按时间判断并切换。

```powershell

param(\[switch]$Force) # -Force 用于强制触发点



\# 基础路径配置 (已更新为你的实际安装路径)

$FanControlExe = "D:\\Program Files (x86)\\FanControl\\FanControl.exe"

$ConfigDir     = "$env:APPDATA\\FanControl\\Configs"

$StateDir      = "C:\\FanControl\_Auto\\state"

$OverrideFlag  = "$StateDir\\override.flag"



$QuietConfig   = "$ConfigDir\\Quiet\_mode.json"

$GameConfig    = "$ConfigDir\\Game\_ultr.json"



New-Item -ItemType Directory -Force -Path $StateDir | Out-Null



\# 判断当前时段应处于什么模式

function Get-TargetConfig {

&#x20;   $min = (Get-Date).Hour \* 60 + (Get-Date).Minute

&#x20;   # Quiet 时段: 12:40(760)-14:00(840), 21:00(1260)-08:00(480)

&#x20;   if (($min -ge 760 -and $min -lt 840) -or ($min -ge 1260) -or ($min -lt 480)) {

&#x20;       return $QuietConfig

&#x20;   } else { return $GameConfig }

}



\# 强制触发点 (只在进入 Quiet 时段的这一分钟强制重置)

$isForcePoint = (((Get-Date).Hour -eq 12 -and (Get-Date).Minute -eq 40) -or 

&#x20;                ((Get-Date).Hour -eq 21 -and (Get-Date).Minute -eq 0))



if ($isForcePoint -or $Force) {

&#x20;   Remove-Item $OverrideFlag -Force -ErrorAction SilentlyContinue

&#x20;   \& $FanControlExe -c $QuietConfig

&#x20;   exit

}



if (Test-Path $OverrideFlag) { exit } # 存在手动免打扰标志，跳过



\& $FanControlExe -c $(Get-TargetConfig)

```



\### 脚本 2：`switch.ps1` (手动干预/热键入口)

用于绑定到快捷方式或 Stream Deck，不仅执行热切换，还会管理免打扰状态。

```powershell

param(\[string]$Mode = "")



\# 基础路径配置 (已更新为你的实际安装路径)

$FanControlExe = "D:\\Program Files (x86)\\FanControl\\FanControl.exe"

$ConfigDir     = "$env:APPDATA\\FanControl\\Configs"

$StateDir      = "C:\\FanControl\_Auto\\state"

$OverrideFlag  = "$StateDir\\override.flag"



$Mode = $Mode.ToLower().Trim()



\# 恢复自动调度

if ($Mode -eq "auto") {

&#x20;   Remove-Item $OverrideFlag -Force -ErrorAction SilentlyContinue

&#x20;   \& "C:\\FanControl\_Auto\\auto\_switch.ps1" -Force # 立即校准当前状态

&#x20;   exit

}



$target = if ($Mode -eq "game") { "$ConfigDir\\Game\_ultr.json" } elseif ($Mode -eq "quiet") { "$ConfigDir\\Quiet\_mode.json" } else { exit }



\# 发送 IPC 消息给运行中的 FanControl 切换配置

\& $FanControlExe -c $target



\# 写入免打扰标志

"$Mode" | Set-Content $OverrideFlag

```

\## 4. 自动化部署：注册定时任务



无需手动去“任务计划程序”里点，直接\*\*以管理员身份打开 PowerShell\*\*，运行以下代码即可一键配置好四个时间点的触发器，并自动绕过 UAC 弹窗：



```powershell

$psExe = "powershell.exe"

$psArg = "-WindowStyle Hidden -ExecutionPolicy Bypass -File `"C:\\FanControl\_Auto\\auto\_switch.ps1`""

$principal = New-ScheduledTaskPrincipal -UserId $env:USERNAME -LogonType Interactive -RunLevel Highest



$tasks = @(

&#x20;   @{ Name = "FanControl-0800(Game)"; Time = "08:00" },

&#x20;   @{ Name = "FanControl-1240(Quiet)"; Time = "12:40" },

&#x20;   @{ Name = "FanControl-1400(Game)"; Time = "14:00" },

&#x20;   @{ Name = "FanControl-2100(Quiet)"; Time = "21:00" }

)



foreach ($t in $tasks) {

&#x20;   $action  = New-ScheduledTaskAction -Execute $psExe -Argument $psArg

&#x20;   $trigger = New-ScheduledTaskTrigger -Daily -At $t.Time

&#x20;   Register-ScheduledTask -TaskName $t.Name -Action $action -Trigger $trigger -Principal $principal -Force

&#x20;   Write-Host "成功注册定时任务: $($t.Name)"

}

```

\## 5. 与外部设备对接 (快捷方式 / Stream Deck)



根据官方建议，你可以将 `switch.ps1` 包装为不同设备的触发命令。



\*\*创建桌面快捷方式 / 键盘宏绑定：\*\*

在桌面上右键 -> 新建快捷方式，分别填入以下目标路径：



\* \*\*切换到游戏模式 (开启免打扰)\*\*:

&#x20; `powershell.exe -WindowStyle Hidden -ExecutionPolicy Bypass -File "C:\\FanControl\_Auto\\switch.ps1" game`

\* \*\*切换到安静模式 (开启免打扰)\*\*:

&#x20; `powershell.exe -WindowStyle Hidden -ExecutionPolicy Bypass -File "C:\\FanControl\_Auto\\switch.ps1" quiet`

\* \*\*清除干预 (恢复自动调度)\*\*:

&#x20; `powershell.exe -WindowStyle Hidden -ExecutionPolicy Bypass -File "C:\\FanControl\_Auto\\switch.ps1" auto`



> \*\*Stream Deck 用户：\*\* 

> 可以使用 "System -> Open" 插件，将 App/File 设为 `powershell.exe`，然后在 Arguments 中填入 `-WindowStyle Hidden -ExecutionPolicy Bypass -File "C:\\FanControl\_Auto\\switch.ps1" game`。



\## 6. 注意事项与排错

1\. \*\*启动路径问题\*\*：官方文档提到建议配置 `Start In (起始位置)`。在本方案中，由于我们在脚本里使用的是\*\*绝对路径\*\* (`D:\\Program Files (x86)\\...` 及 `$env:APPDATA\\...`)，因此完全避免了路径识别错误的问题，比官方快捷方式更稳健。

2\. \*\*FanControl 必须保持后台运行\*\*：脚本利用了官方 `-c` 参数的 IPC 通信功能，因此 FanControl 本身应设置为开机自启并常驻托盘。

3\. \*\*UAC 提示\*\*：使用快捷方式执行手动切换时，由于不经过任务计划程序，\*\*可能\*\*会触发一次 UAC 提示。如果希望连手动切换也彻底静默无弹窗，可以通过“任务计划程序”创建由最高权限运行的手动 Task，并在快捷方式中使用 `schtasks /run /tn "TaskName"` 来调用。
