# FanControl 持续监控系统使用说明

**创建时间：** 2026-04-12 13:10

---

## 📊 功能说明

### 持续监控系统

**工作原理：**
1. 每 **5秒** 采集一次状态数据
2. 每 **5分钟** 自动总结并保存
3. 生成 JSON 和 Markdown 格式报告
4. 按 Ctrl+C 停止时自动保存最终总结

---

## 🚀 使用方法

### 方法 1：前台运行（推荐）

```powershell
# 使用默认参数（5秒采集，5分钟总结）
.\monitor_simple.ps1

# 自定义参数（10秒采集，1分钟总结）
.\monitor_simple.ps1 -IntervalSeconds 10 -SummaryMinutes 1
```

**显示：**
```
Monitoring started... (Ctrl+C to stop)
[13:07:26] Process:OK Config:Quiet_mode.json Match:OK Samples:1
[13:07:36] Process:OK Config:Quiet_mode.json Match:OK Samples:2
...
Saving summary...
Saved: monitor_20260412_130826.json
```

---

### 方法 2：后台运行

```powershell
# 后台运行（使用默认参数）
Start-Process powershell -ArgumentList "-WindowStyle Hidden -ExecutionPolicy Bypass -File `"C:\FanControl_Auto\monitor_simple.ps1`""
```

---

### 方法 3：创建计划任务（开机自动监控）

```powershell
# 创建计划任务
$action = New-ScheduledTaskAction -Execute "powershell.exe" -Argument "-WindowStyle Hidden -ExecutionPolicy Bypass -File `"C:\FanControl_Auto\monitor_simple.ps1`""
$trigger = New-ScheduledTaskTrigger -AtStartup -RandomDelay (New-TimeSpan -Minutes 3)
$principal = New-ScheduledTaskPrincipal -UserId $env:USERNAME -LogonType Interactive -RunLevel Highest
Register-ScheduledTask -TaskName "FanControl-Monitor" -Action $action -Trigger $trigger -Principal $principal -Force
```

### 数据字段说明

|------|------|--------|

| ConfigMatch | 配置是否匹配 | true / false |

## 📊 监控数据解读

### 1. 进程在线率

**计算方式：**
```
在线率 = (ProcessRunning=true 的样本数 / 总样本数) × 100%
```

**示例：**
```
总样本数: 30
运行样本数: 29
在线率: 96.67%
```

---

### 2. 配置匹配率

**计算方式：**
```
匹配率 = (ConfigMatch=true 的样本数 / 总样本数) × 100%
```

**示例：**
```
总样本数: 30
匹配样本数: 30
匹配率: 100%
```

---

### 3. 状态变化检测

监控可以检测到：
- ✅ 进程启动/停止
- ✅ 配置切换
- ✅ 配置不匹配
- ✅ Override 状态变化

## 📈 数据分析示例

### 示例：读取最近一次监控数据

```powershell
# 获取最新的监控文件
$latestFile = Get-ChildItem "C:\FanControl_Auto\monitor_data\*.json" | Sort-Object LastWriteTime -Descending | Select-Object -First 1

# 读取数据
$data = Get-Content $latestFile.FullName | ConvertFrom-Json

# 分析
$runningCount = ($data | Where-Object { $_.ProcessRunning }).Count
$matchCount = ($data | Where-Object { $_.ConfigMatch }).Count
$total = $data.Count

Write-Host "Analysis Report:"
Write-Host "  Total Samples: $total"
Write-Host "  Process Uptime: $([math]::Round(($runningCount / $total) * 100, 2))%"
Write-Host "  Config Match Rate: $([math]::Round(($matchCount / $total) * 100, 2))%"
```

---

### 示例：查找配置不匹配的时间点

```powershell
$data | Where-Object { -not $_.ConfigMatch } | Select-Object Timestamp, CurrentConfig, ExpectedConfig
```

---

### 示例：统计配置切换事件

```powershell
$switches = @()
for ($i = 1; $i -lt $data.Count; $i++) {
    if ($data[$i].CurrentConfig -ne $data[$i-1].CurrentConfig) {
        $switches += [PSCustomObject]@{
            Time = $data[$i].Timestamp
            From = $data[$i-1].CurrentConfig
            To = $data[$i].CurrentConfig
        }
    }
}
$switches | Format-Table -AutoSize
```

## 📝 总结文件命名规则

**格式：** `monitor_YYYYMMDD_HHMMSS.json`

**示例：**
- `monitor_20260412_130826.json` - 2026年4月12日 13:08:26 生成
- `monitor_20260412_135826.json` - 2026年4月12日 13:58:26 生成

**解析：**
- YYYY = 年份
- MM = 月份
- DD = 日期
- HH = 小时
- MM = 分钟
- SS = 秒

### 3. 异常检测

编写脚本检测异常：
```powershell
$data = Get-Content "latest_monitor.json" | ConvertFrom-Json

# 检测进程异常停止
$notRunning = $data | Where-Object { -not $_.ProcessRunning }
if ($notRunning.Count -gt 0) {
    Write-Host "Warning: Process stopped at:" -ForegroundColor Red
    $notRunning | Select-Object Timestamp
}

# 检测配置不匹配
$mismatch = $data | Where-Object { -not $_.ConfigMatch }
if ($mismatch.Count -gt 0) {
    Write-Host "Warning: Config mismatch at:" -ForegroundColor Yellow
    $mismatch | Select-Object Timestamp, CurrentConfig, ExpectedConfig
}
```
