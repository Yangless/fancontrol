## 📊 监控机制总览

### 三层监控体系

```
┌─────────────────────────────────────────┐
│  实时状态文件 (current_status.json)      │
│  - 记录每次切换的目标配置                │
│  - 记录实际生效的配置                    │
│  - 记录验证结果（是否匹配）              │
└─────────────────────────────────────────┘
         ↓
┌─────────────────────────────────────────┐
│  详细日志文件 (auto_switch.log)          │
│  - 记录每个操作步骤                      │
│  - 记录等待时间                          │
│  - 记录验证过程                          │
└─────────────────────────────────────────┘
         ↓
┌─────────────────────────────────────────┐
│  系统托盘通知                            │
│  - 成功/失败即时通知                     │
│  - 无需手动查看日志                      │
└─────────────────────────────────────────┘
```

## ✅ 验证机制

### 1. 切换后自动验证

脚本执行配置切换后，会：
```powershell
1. 执行 FanControl.exe -c <Config>
2. 等待 CACHE 文件更新（最长10秒）
3. 读取 CACHE 文件，检查 CurrentConfigFileName
4. 对比目标配置和实际配置
5. 记录验证结果到状态文件
```

---

### 2. 验证结果存储

**状态文件：** `C:\FanControl_Auto\state\current_status.json`

```json
{
  "Timestamp": "2026-04-11 23:36:35",
  "TargetConfig": "Quiet_mode.json",
  "ActualConfig": "Quiet_mode.json",
  "Status": "SUCCESS",
  "Verified": true,
  "ProcessRunning": true,
  "ProcessId": 2144,
  "Message": "Switch to Quiet_mode.json - Verified"
}
```

**字段说明：**

| 字段 | 说明 | 示例值 |
|------|------|--------|
| Timestamp | 切换时间 | 2026-04-11 23:36:35 |
| TargetConfig | 目标配置文件 | Quiet_mode.json |
| ActualConfig | 实际生效配置 | Quiet_mode.json |
| Status | 切换状态 | SUCCESS / FAILED |
| Verified | 是否验证通过 | true / false |
| ProcessRunning | FanControl 是否运行 | true / false |
| ProcessId | 进程 ID | 2144 |
| Message | 详细信息 | Switch to Quiet_mode.json - Verified |

---

### 3. 如何判断切换是否成功

**方法 1：查看状态文件**

```powershell
cat C:\FanControl_Auto\state\current_status.json
```

**判断标准：**
- ✅ `"Status": "SUCCESS"` → 切换成功
- ✅ `"Verified": true` → 配置验证通过
- ✅ `"TargetConfig" == "ActualConfig"` → 配置匹配
- ❌ `"Status": "FAILED"` → 切换失败
- ❌ `"Verified": false` → 配置未验证

## 🔍 验证逻辑详解

### 配置切换验证流程

```
1. 执行切换
   └─> FanControl.exe -c Game.json -tray

2. 等待 CACHE 更新（最长10秒）
   ├─> 第1秒：读取 CACHE
   │   └─> CurrentConfigFileName = ?
   ├─> 第2秒：再次读取
   │   └─> CurrentConfigFileName = ?
   └─> ...
       └─> 第N秒：CurrentConfigFileName = Game.json ✅

3. 验证结果
   ├─> 目标 == 实际 → SUCCESS (Verified: true)
   └─> 目标 != 实际 → FAILED (Verified: false)

4. 更新状态文件
   └─> current_status.json

5. 显示通知
   ├─> SUCCESS → "Successfully switched to Game.json"
   └─> FAILED → "Switch to Game.json may have failed"
```

## ⚠️ 故障排查

### 问题 1：切换后显示 FAILED

**原因：**
- FanControl 未启动
- 配置文件损坏
- CACHE 文件未更新

**排查步骤：**

```powershell
# 1. 检查进程
Get-Process -Name FanControl

# 2. 检查配置文件
ls "D:\Program Files (x86)\FanControl\Configurations\"

# 3. 检查 CACHE 文件
cat "D:\Program Files (x86)\FanControl\Configurations\CACHE"

# 4. 手动启动并验证
Stop-Process -Name FanControl -Force
.\auto_switch.ps1
.\check_status.ps1
```

---

### 问题 2：状态文件显示 Verified: false

**原因：**
- 切换超时（超过10秒）
- 配置文件名称不匹配

**排查：**

```powershell
# 查看详细日志
cat C:\FanControl_Auto\logs\auto_switch.log | Select-String "Waiting"

# 检查配置名称是否正确
cat "D:\Program Files (x86)\FanControl\Configurations\CACHE" | ConvertFrom-Json | Select-Object CurrentConfigFileName
```

---

### 问题 3：日志未更新

**原因：**
- 脚本未执行
- 权限问题

**排查：**

```powershell
# 检查任务计划
Get-ScheduledTask | Where-Object { $_.TaskName -like 'FanControl*' }

# 手动触发测试
powershell -ExecutionPolicy Bypass -File "C:\FanControl_Auto\auto_switch.ps1"

# 检查日志目录权限
icacls C:\FanControl_Auto\logs
```
