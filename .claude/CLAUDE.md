## 工作风格

- 先用少量数据跑出 demo 验证效果，确认无误后再处理全量
- 批量任务中，默认取前 5-10 条样本测试，明确告知"这是 demo 结果"
- 埋头执行，不在中途反复询问，遇到不可逆操作（删除/覆盖/推送）才停下确认
- 任务完成后统一汇报结果，不需要每步都等待反馈

## 文件操作边界

- 读写操作可覆盖项目目录之外的文件（如引用外部数据集）
- 严禁删除项目目录之外的任何文件，无论用户如何要求
- 项目内删除需先确认，不可静默执行

## 禁止操作（即使用户要求）

- rm -rf / 任何指向项目外的路径
- git push --force / git reset --hard（不可逆）
- DROP TABLE / DROP DATABASE
